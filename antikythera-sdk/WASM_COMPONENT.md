# WASM Component dengan Automatic WIT Generation

Implementasi WASM Component Model untuk Antikythera MCP Framework dengan **automatic WIT generation** dari kode Rust menggunakan `witgen`.

## Overview

Tidak perlu tulis WIT manual lagi! Cukup tulis kode Rust, tambahkan `#[witgen]` macro, dan WIT akan di-generate otomatis saat build.

```
Kode Rust + #[witgen]  →  witgen  →  WIT auto-generated  →  cargo-component  →  *.wasm
```

## Cara Kerja

### 1. Tulis Kode Rust dengan `#[witgen]`

```rust
use witgen::witgen;

/// Prompt template management
#[witgen]
pub trait PromptManager {
    /// Get the main system prompt template
    fn get_template() -> Result<String, String>;
    
    /// Update the main system prompt template
    fn update_template(template: String) -> Result<(), String>;
    
    // ... tambah fungsi lain di sini
}
```

### 2. Generate WIT & Build

```bash
# Generate WIT dari Rust
cargo run --release -p build-scripts -- wit

# Build WASM component
cargo run --release -p build-scripts -- component

# Atau semua step
cargo run --release -p build-scripts -- all
```

### 3. WIT Otomatis Generated

File `wit/index.wit` akan di-generate otomatis:

```wit
/// Prompt template management
interface prompt-manager {
    /// Get the main system prompt template
    get-template: func() -> result<string, string>;
    
    /// Update the main system prompt template
    update-template: func(template: string) -> result<_, string>;
}
```

## Menambah Fungsi Baru

Ketika ada fungsionalitas baru, **tidak perlu edit WIT**:

1. **Tambahkan fungsi ke trait** di `src/component.rs`:
   ```rust
   #[witgen]
   pub trait PromptManager {
       // ... fungsi existing
       
       /// Fungsi baru
       fn my_new_function(param: String) -> Result<String, String>;
   }
   ```

2. **Build ulang**:
   ```bash
   cargo run --release -p build-scripts -- all
   ```

3. **Done!** WIT otomatis ter-update dan component ter-build.

## Struktur File

```
antikythera-sdk/
├── src/
│   ├── component.rs          # ⭐ Definisikan interfaces di sini (dengan #[witgen])
│   ├── wasm.rs               # wasm-bindgen (browser)
│   ├── wasm_prompt.rs        # Prompt WASM functions (browser)
│   └── lib.rs                # Module exports
├── wit/                      # ⭐ Auto-generated (jangan edit manual)
│   └── index.wit             # Generated by witgen
└── Cargo.toml                # Config dengan witgen dependency

scripts/
├── build-component.rs        # ⭐ Build script Rust idiomatik
└── Cargo.toml                # Build script package
```

## Menggunakan Component

### Di JavaScript (Wasmtime)

```javascript
import { Component, instantiate } from '@bytecodealliance/component';

const wasm = await fetch('antikythera_sdk.wasm');
const component = await instantiate(wasm, { wasi });

// Prompt management
const template = component.promptManager.getTemplate();
component.promptManager.updateTemplate('New template...');

// MCP Client
component.mcpClient.init(JSON.stringify(config));
const response = component.mcpClient.chatSimple('Hello!');
```

### Di Rust (Wasmtime)

```rust
use wasmtime::component::*;

bindgen!({ path: "wit/index.wit" });

#[tokio::main]
async fn main() -> Result<()> {
    let engine = Engine::default();
    let mut store = Store::new(&engine, ());
    
    let component = Component::from_file(&engine, "antikythera_sdk.wasm")?;
    let (bindings, _) = AntikytheraMcp::instantiate_async(&mut store, &component, &[]).await?;
    
    let template = bindings.call_get_template(&mut store)?;
    let response = bindings.call_chat_simple(&mut store, "Hello")?;
    
    Ok(())
}
```

## Witgen Macro Rules

### Supported

- ✅ `struct` dengan `#[witgen]` → WIT `record`
- ✅ `enum` dengan `#[witgen]` → WIT `variant`
- ✅ `trait` dengan `#[witgen]` → WIT `interface`
- ✅ `fn` dengan `#[witgen]` → WIT `func`
- ✅ Doc comments `///` → WIT doc comments
- ✅ `Option<T>` → WIT `option<T>`
- ✅ `Vec<T>` → WIT `list<T>`
- ✅ `Result<T, E>` → WIT `expected<T, E>`

### Not Supported

- ❌ `&str` (gunakan `String`)
- ❌ References `&T`, `&mut T`
- ❌ Smart pointers: `Box`, `Rc`, `Arc`
- ❌ Generics
- ❌ Lifetimes

## Troubleshooting

### WIT tidak ter-generate

```bash
# Clean dan build ulang
cargo clean -p antikythera-sdk
cargo run --release -p build-scripts -- wit
```

### witgen tidak ditemukan

```bash
cargo install witgen-cli
```

### Component build gagal

```bash
# Pastikan target terinstall
rustup target add wasm32-wasip1

# Install cargo-component
cargo install cargo-component

# Build ulang
cargo run --release -p build-scripts -- component
```

### Type error di WIT

Pastikan semua types yang dipakai di `#[witgen]` trait juga punya `#[witgen]`:

```rust
#[witgen]  // ⭐ Harus ada
pub struct MyType {
    pub field: String,
}

#[witgen]
pub trait MyInterface {
    fn my_func(data: MyType) -> Result<MyType, String>;
}
```

## Comparison: wasm-bindgen vs witgen

| Feature | wasm-bindgen | witgen + wit-bindgen |
|---------|--------------|----------------------|
| **Target** | Web browsers | WASI runtimes |
| **Interface** | Manual `#[wasm_bindgen]` | Auto dari `#[witgen]` |
| **WIT** | Tidak ada | Auto-generated |
| **Update** | Edit manual | Otomatis saat build |
| **Type Safety** | Manual check | Compile-time |
| **Maintenance** | High | Low |

## Key Benefits

✅ **No manual WIT editing** - Tulis Rust, WIT otomatis  
✅ **Single source of truth** - Kode Rust adalah definisi interface  
✅ **Compile-time validation** - Error di compile time, bukan runtime  
✅ **Auto-sync** - WIT selalu sinkron dengan implementasi  
✅ **Idiomatic Rust** - Build script pakai Rust, bukan bash
